HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • meta elements contain information about the page that is used by the browser.
  • The charset element sets our character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The heading (h1, h2 etc) elements represent the level of heading a given text block represents.
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list items.
  • The img element contains information about images that are displayed on the webpage.
  • The img element can contain various attributes; the src attribute is the most important because it defines the location of the image.
  • The br element creates a line of empty space, or a line break, between two blocks of content.
  • The a element (which stands for anchor) creates links to the same webpage or other webpages.
    The href attribute points to the URL for the link.

CSS

  • CSS is a programming language used by developers to define how webpage content is presented to users:
    colours, aesthetics, visual layout, and organization.
  • inline css is used to style an element driectly in the HTML file.
  • An external css style sheet alows developers to add CSS rules in a separate file, which makes design changes easier.
  • When using an external style sheet, we link the file using a link element placed in the HTML's head.
  • The margin property indicates how much space we want around the outside of an element.
  • The padding property adds space around the content inside an element.
  • The * symbol applies a rule to all the elements visible on the page.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • The add command adds modifications in the current working branch to the staging area.
    The -A flag indicates that we want to add all changes.
  • The pull command is used to receive a branch's modifications into the local environment.
  • The push command pushes the changes we have locally to our remote GitHub branch.

JavaScript

  • In JavaScript, to declare a variable, we use the var keyword.
  • After the var keyword, we give the variable a name inside "...", this must be unique.
  • We use an assignment operator = to assign the specific data that we want stored in our variable.
  • Each line of JavaScript code ends with ;
  • The JS file is linked within the body element.
  • Control flow is the order in which a computer executes code in a script.
  • An if statement will check to see if the condition that we provide is truthy.
    if not truthy, then the computer will ignore whatever is in between the curly brackets and move on.
  • else if is used if there is more than one condition.
  • An array is a single variable that is used to hold a group of data; typically used to hold data that is related in some way.
  • The location of an item in an array is identified with a unique number called an index.
  • In JavaScript and many other programming languages, arrays are zero-indexed and sequential.
  • /* followed by *\ comments out multiples lines of code.
